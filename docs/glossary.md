# Glossary

Project terminology, defined in plain business language. Use these terms consistently across agent prompts and documentation.

**Agent** — A specialized AI component with a single defined job, an explicit set of inputs, and an explicit set of outputs. Agents are connected into a pipeline; no agent does another agent's job.

**Analyst** — An agent whose job is to produce evidence about one domain (market regime, price action, or options pricing). Analysts describe what is; they never recommend trades.

**Strategist** — The agent that synthesizes the analysts' evidence into a concrete trade proposal, or concludes that no trade is warranted.

**Peer Review** — The independent challenge a proposal must survive before it can advance. The reviewer's job is to find reasons the proposal fails — unstated assumptions, evidence it contradicts, risks it understates.

**Approved Strategy** — A proposal that has survived peer review. Only approved strategies reach the Portfolio Manager; approval does not guarantee acceptance.

**Recommendation** — The strategist's output for a candidate: either a fully specified strategy proposal with its supporting evidence, or a "no trade" conclusion.

**No Trade** — A first-class outcome in which the pipeline concludes the candidate does not merit a position. Not a failure of the process, but the process working as a filter.

**Watchlist** — The trader's curated list of candidate symbols and their upcoming earnings events; the entry point of the pipeline.

**Expected Move** — The size of the price change the options market is pricing in around an event, inferred from option prices. A benchmark against which a trade's thesis is judged.

**Edge** — A specific, stated reason to believe a trade is favorably priced relative to its risk. An edge must be supported by evidence; "I think it goes up" is not an edge.

**Risk-Adjusted Return** — The value of a trade judged relative to the risk taken to earn it, not by potential profit alone. The lens through which proposals and portfolio decisions are evaluated.

**Evidence** — An observable input: market data, a prior stage's written output, or a stated fact. Every claim in the pipeline must trace to evidence or be labeled an assumption.

**Assumption** — Something a conclusion depends on that is not supported by available evidence. Assumptions are permitted only when explicitly labeled as such in the written record.
